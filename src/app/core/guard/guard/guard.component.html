CanActivate

CanActivate is used to control whether a user can access a specific route. It checks conditions like authentication or authorization before allowing navigation.

CanActivateChild

CanActivateChild is used to control access to all child routes under a parent route. It helps avoid repeating the same guard logic for multiple child routes.

CanDeactivate

CanDeactivate is used to prevent users from leaving a route when there are unsaved changes. It runs before navigating away from a page.

CanLoad

CanLoad is used to prevent lazy-loaded modules from being loaded if the user is not authorized. It improves security and application performance.

Resolve

Resolve is used to fetch required data before a route is activated. The page loads only after the data is successfully retrieved.

CanMatch

CanMatch is used to decide which route configuration should be matched before navigation. It is useful when multiple routes share the same path.
